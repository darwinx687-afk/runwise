# Project Constitution

Runwise exists to make AI agent, MCP server, and LLM application readiness visible, repeatable, and auditable.

## Principles

1. Local first: the core toolkit must work without hosted services.
2. TypeScript first: public package boundaries should expose typed contracts.
3. Evidence over assertion: checks, traces, replays, and evals should produce reviewable artifacts.
4. Open boundaries: integrations must be adapters, not hidden runtime ownership.
5. Bilingual project surface: core project documentation should be available in English and Simplified Chinese.

## Phase 0 Scope

Phase 0 creates repository foundations only. It may define package boundaries, docs, governance files, and a CLI placeholder.

## Explicit Boundaries

Phase 0 must not add login, cloud accounts, billing, databases, complex dashboard logic, or unrelated AI agent runtime behavior.
