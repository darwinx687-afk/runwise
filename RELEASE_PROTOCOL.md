# Release Protocol

Runwise maintains an initial open-source preview. This protocol covers repository readiness, preview tags, GitHub prerelease verification, and package publishing boundaries.

## Pre-release Checklist

- `pnpm install` completes.
- `pnpm check` passes.
- `pnpm test` passes.
- `RUN_STATE.md` is current.
- `DECISION_LOG.md` captures structural decisions.
- README files are mirrored in English and Simplified Chinese.
- `CHANGELOG.md` and launch notes are current.
- `.runwise/` generated artifacts are ignored and untracked.

## Versioning

The first release candidate uses `0.1.0-preview.0`.

Package metadata remains source-first in this preview. npm publishing metadata should be finalized only in a later publishing loop.

## First Tag Preparation

The first preview tag is `v0.1.0-preview.0`.

These were the prepared tag commands:

```bash
git status --short
pnpm check
pnpm check:types
pnpm test
pnpm exec runwise doctor
git tag -a v0.1.0-preview.0 -m "Runwise v0.1.0-preview.0"
git push origin v0.1.0-preview.0
```

Do not move, delete, or force-update the tag after it is pushed.

Before creating the tag, verify GitHub Actions is green on `main`, generated `.runwise/` artifacts are still ignored, and npm publishing plus GitHub Marketplace release remain deferred.

## Preview GitHub Release Creation

The tag `v0.1.0-preview.0` already exists. The GitHub Release was created as a prerelease in Loop 10H after the prerelease creation loop was explicitly approved.

Final release notes should come from `docs/GITHUB_RELEASE_NOTES_v0.1.0-preview.0.md`.

Historical command shape:

```bash
gh release edit v0.1.0-preview.0 \
  --repo darwinx687-afk/runwise \
  --title "Runwise v0.1.0-preview.0" \
  --notes-file docs/GITHUB_RELEASE_NOTES_v0.1.0-preview.0.md \
  --prerelease \
  --latest=false
```

Verification commands:

```bash
gh release view v0.1.0-preview.0 --repo darwinx687-afk/runwise
gh release list --repo darwinx687-afk/runwise --limit 5
```

npm publishing remains deferred. GitHub Marketplace release remains deferred. After release creation, verify the GitHub release page renders correctly.

## Post-prerelease verification

- Release exists at https://github.com/darwinx687-afk/runwise/releases/tag/v0.1.0-preview.0.
- Release is marked as a prerelease and is not a draft.
- Tag `v0.1.0-preview.0` remains unchanged and must not be moved or recreated.
- npm publishing remains deferred.
- GitHub Marketplace release remains deferred.
- Release notes are reviewed and loaded from `docs/GITHUB_RELEASE_NOTES_v0.1.0-preview.0.md`.
- Source archives are auto-generated by GitHub.
- No generated `.runwise/` artifacts are attached to the release.
