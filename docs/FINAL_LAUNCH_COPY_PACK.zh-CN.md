## V2EX / 掘金发布稿

我在做一个本地优先的 AI Agent readiness / trace replay / eval 工具，叫 Runwise。第一个 public preview `v0.1.0-preview.0` 已发布。

目前可以做这些事：

- `runwise doctor` 生成本地就绪度报告
- 输出 JSON、Markdown 和静态 HTML 报告
- 验证本地 `runwise.agent_trace` 文件
- 从 trace 生成静态 replay 报告
- 从失败或高风险 trace 生成 Failure-to-Eval 草稿
- 检测 MCP、RAG、browser agent、OpenAI-compatible API 和国内大模型服务商等本地兼容性信号

当前版本只支持源码安装。npm、Marketplace、hosted SaaS、cloud sync 和官方集成声明都没有包含。

希望收到关于 Doctor 误报/漏报、trace replay 可读性、Failure-to-Eval 输出结构和国内大模型检测信号的反馈。

Release: https://github.com/darwinx687-afk/runwise/releases/tag/v0.1.0-preview.0

## 面向国内企业 AI 落地的介绍

Runwise 面向正在落地 AI Agent、MCP server、RAG 和大语言模型应用的团队，目标是在 demo、CI 门禁和发布审查前生成可审阅的本地证据。

第一个公开预览版本包含本地就绪度检查、报告生成、trace 验证、静态 replay、Failure-to-Eval 生成和国内大模型服务商兼容性信号检测。

当前版本不包含 npm 发布、hosted SaaS、cloud sync、登录、数据库、telemetry、模型调用或官方生态合作声明。所有输出都以本地、可复现、可审阅的文件为主。

Release: https://github.com/darwinx687-afk/runwise/releases/tag/v0.1.0-preview.0
