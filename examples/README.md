# Runwise Examples

These examples are lightweight compatibility examples and fixtures. They are not production AI apps, do not include real dependencies, and should not contain secrets.

## Folders

- `mcp-demo`: MCP detection signals and high-risk tool approval documentation.
- `rag-demo`: prompt, trace, and eval placeholders for grounded-answer workflows.
- `browser-agent-demo`: browser-agent compatibility notes and browser-use style signals.
- `enterprise-workflow-demo`: Dify-style workflow platform detection signals.
- `china-ready-llm-demo`: OpenAI-compatible API and China-ready LLM provider configuration placeholders.
- `codex-project-demo`: Codex, Claude Code, Cursor, and Windsurf style project instruction signals.
- `traces`: local `runwise.agent_trace` fixtures for validation, replay, and Failure-to-Eval.

## Safety

Examples are designed for Runwise detection and documentation. They do not run models, execute tools, call external APIs, or start external frameworks.
